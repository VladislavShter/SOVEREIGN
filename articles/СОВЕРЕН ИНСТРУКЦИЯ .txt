     ПОДГОТОВКА СРЕДЫ РАЗРАБОТКИ — ЭТАП 1

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━


   Цель этапа — получить полностью рабочее окружение для написания
   Android-приложения Sovereign на Kotlin, с возможностью запуска
   на эмуляторе и реальном устройстве.

   Время выполнения: 2-3 часа (с учётом скачивания).
   Диск: потребуется ~15-20 ГБ.

   Идём пошагово. После каждого шага ты пишешь «дальше»,
   и мы переходим к следующему.

  ШАГ 1  ─  ПРОВЕРКА ВИРТУАЛИЗАЦИИ В BIOS


  Intel VT-x должен быть включён, иначе эмулятор Android
  не запустится. Поскольку у тебя iMac на Windows — проверяем.

  Действия:
  
  1.1  Открыть «Диспетчер задач» (Ctrl + Shift + Esc)
  1.2  Перейти на вкладку «Производительность»
  1.3  Выбрать «ЦП» слева
  1.4  Справа внизу найти строку «Виртуализация»

  Если написано «Включено» — отлично, идём дальше.
  Если «Отключено» — нужно зайти в BIOS и включить
  Intel Virtualization Technology (VT-x). Я подскажу как
  для iMac на Boot Camp, если потребуется.

  ШАГ 2  ─  УСТАНОВКА ANDROID STUDIO


  Это основная среда разработки. Бесплатная, официальная,
  от Google. Включает в себя редактор кода, отладчик,
  эмулятор Android, и менеджер SDK.

  Действия:
  
  2.1  Скачать дистрибутив:
       https://developer.android.com/studio
       Кнопка «Download Android Studio» — она сама определит
       что у тебя Windows и предложит правильный файл.
       Размер: ~1.2 ГБ.

  2.2  Запустить установщик. Соглашаемся со всеми
       значениями по умолчанию. Не меняем папку установки.

  2.3  При первом запуске:
       —  Выбрать «Do not import settings»
       —  Согласиться с отправкой анонимной статистики
          (или отказаться — на твой выбор)
       —  В «Welcome» нажать «Next»
       —  Тип установки: «Standard» (обычный)
       —  Выбрать цветовую тему: «Darcula» (тёмная) или
          «IntelliJ Light» — на твой вкус
       —  В «Verify Settings» нажать «Next»
       —  Принять все лицензионные соглашения
       —  Нажать «Finish»

  2.4  Android Studio начнёт загружать Android SDK
       (~5-8 ГБ). Это займёт 20-60 минут в зависимости
       от скорости интернета. Можно идти пить чай.

  ШАГ 3  ─  РУСИФИКАЦИЯ ANDROID STUDIO


  Android Studio изначально на английском, но есть
  официальный русский язык-пак от JetBrains.

  Действия:
  
  3.1  В Android Studio открыть Settings:
       File → Settings (или Ctrl + Alt + S)

  3.2  Слева найти «Plugins»

  3.3  Сверху выбрать вкладку «Marketplace»

  3.4  В поиске ввести: Russian Language Pack

  3.5  Найти плагин от JetBrains s.r.o. — нажать «Install»

  3.6  После установки появится кнопка «Restart IDE» —
       нажать, IDE перезапустится на русском.


  Важно: интерфейс будет на 80-90% на русском. Некоторые
  технические термины и сообщения об ошибках останутся
  на английском — это нормально, так устроен плагин.

  ШАГ 4  ─  УСТАНОВКА ДОПОЛНИТЕЛЬНЫХ SDK


  Нам нужны Android API уровни от 30 до 35.
  По умолчанию Android Studio ставит только самый свежий.

  Действия:
  
  4.1  В Android Studio открыть SDK Manager:
       Tools → SDK Manager
       (на русском: «Инструменты» → «Менеджер SDK»)

  4.2  Вкладка «SDK Platforms» — отметить галочками:
       —  Android 11 (API 30)
       —  Android 12 (API 31)
       —  Android 13 (API 33)
       —  Android 14 (API 34)
       —  Android 15 (API 35) — должен быть уже установлен

  4.3  Вкладка «SDK Tools» — убедиться что отмечены:
       —  Android SDK Build-Tools (последняя версия)
       —  Android Emulator
       —  Android SDK Platform-Tools
       —  Intel x86 Emulator Accelerator (HAXM installer)

  4.4  Нажать «Apply» — пойдёт скачивание (~3-5 ГБ).
       Принять все лицензии когда попросит.

  ШАГ 5  ─  СОЗДАНИЕ ЭМУЛЯТОРА ANDROID


  Виртуальное Android-устройство для тестирования
  без физического телефона.

  Действия:
  
  5.1  В Android Studio открыть Device Manager:
       Tools → Device Manager
       (на русском: «Инструменты» → «Менеджер устройств»)

  5.2  Нажать «Create Virtual Device» (Создать)

  5.3  Выбрать модель:
       —  Категория: Phone
       —  Pixel 7 (рекомендую — это эталонный Android)
       —  Нажать «Next»

  5.4  Выбрать системный образ:
       —  Vanilla Ice Cream (API 35) или
       —  UpsideDownCake (API 34)
       —  Если рядом есть кнопка «Download» — нажать,
          образ скачается (~1.5 ГБ)
       —  После скачивания нажать «Next»

  5.5  Имя устройства: оставить по умолчанию или назвать
       «Sovereign Test Device»

  5.6  Нажать «Finish». Эмулятор появится в списке.

  5.7  Запустить эмулятор кнопкой ▶ для проверки.
       Первый запуск: 1-3 минуты. Должен появиться
       виртуальный Android-телефон.

  ШАГ 6  ─  УСТАНОВКА GIT (ЛОКАЛЬНО)


  Версионный контроль на твоём компьютере.
  Никаких пушей в облако — только локально.

  Действия:
  
  6.1  Скачать Git для Windows:
       https://git-scm.com/download/win
       Должен начаться автоматический download.
       Размер: ~60 МБ.

  6.2  Запустить установщик. На всех экранах
       нажимать «Next» с настройками по умолчанию.
       Это безопасно, ничего не сломается.

  6.3  Проверить установку: открыть командную строку
       (Win + R, ввести cmd) и выполнить:
       
            git --version
       
       Должна появиться строка вроде:
       git version 2.43.0.windows.1


  Опционально: GitHub Desktop как графическая оболочка
  для Git. Полезен если не любишь командную строку.
  Скачивается тут:
       https://desktop.github.com/

  Используется только для **локальной** работы с Git,
  без публикации на GitHub.com.

  ШАГ 7  ─  ПОДГОТОВКА ФИЗИЧЕСКОГО ANDROID-УСТРОЙСТВА


  Любая разработка для Android требует физического
  устройства для финальной отладки. Эмулятор хорош
  для базы, но реальные баги ловятся только на железе.

  Действия:
  
  7.1  На своём Android-телефоне:
       Настройки → О телефоне → Номер сборки
       
       Нажать 7 раз подряд. Появится сообщение
       «Вы стали разработчиком».

  7.2  Вернуться в Настройки → найти «Для разработчиков»
       (на разных производителях лежит в разных местах:
       Дополнительно, Система, или прямо в корне).

  7.3  Включить:
       —  «Отладка по USB» (USB debugging)
       —  «Беспроводная отладка» (Wireless debugging) —
          если есть, Android 11+

  7.4  Подключить телефон к компьютеру USB-кабелем
       (используем оригинальный, не дешёвый).

  7.5  На телефоне появится запрос «Разрешить отладку?» —
       нажать «Разрешить», поставить галочку
       «Всегда разрешать с этого компьютера».

  7.6  В Android Studio в выпадающем списке вверху
       (рядом с кнопкой ▶) должен появиться твой телефон.
       Это значит подключение работает.


  Какой у тебя телефон — напиши мне, я подскажу
  специфичные настройки если они потребуются
  (Xiaomi, Honor, OPPO имеют особенности).

  ШАГ 8  ─  СОЗДАНИЕ ПРОЕКТА «SOVEREIGN»


  Создаём пустой Android-проект с правильной
  структурой под нашу архитектуру.

  Действия:
  
  8.1  Android Studio → File → New → New Project
       (или «Создать новый проект» на стартовом экране)

  8.2  Выбрать шаблон: «Empty Activity»
       Нажать «Next»

  8.3  Параметры проекта:
       —  Name: Sovereign
       —  Package name: pw.s0vereign.sovereign
       —  Save location: выбрать удобную папку,
          например D:\Projects\Sovereign
       —  Language: Kotlin
       —  Minimum SDK: API 30 ("R", Android 11)
       —  Build configuration language: Kotlin DSL

  8.4  Нажать «Finish»

  8.5  Android Studio будет настраивать проект 3-5 минут
       (сборка Gradle, скачивание зависимостей).
       После этого появится готовый проект.

  ШАГ 9  ─  ПЕРВЫЙ ЗАПУСК «HELLO SOVEREIGN»


  Проверяем что вся цепочка работает —
  от кода до запуска на устройстве.

  Действия:
  
  9.1  В верхней панели выбрать устройство —
       либо эмулятор «Pixel 7», либо твой телефон.

  9.2  Нажать зелёную ▶ (Run).

  9.3  Подождать 1-3 минуты. Проект должен собраться,
       установиться на устройство и запуститься.

  9.4  На экране должно появиться приложение
       с надписью «Hello Android!» или подобной.


  Если запустилось — поздравляю, мы готовы.
  Дальше начинаем писать код Sovereign.

  ОЖИДАЕМЫЕ ПРОБЛЕМЫ НА ЭТАПЕ 1


  ◈  «HAXM installation failed» при настройке эмулятора —
     означает что виртуализация в BIOS выключена.
     Возвращаемся к Шагу 1.
  
  ◈  Эмулятор очень медленно загружается — это нормально
     для твоего i5-7500. Дай 3-5 минут на первый запуск,
     потом будет быстрее.
  
  ◈  Android Studio «зависает» при первой загрузке —
     это Gradle качает зависимости. До 30 минут это
     нормально. Не закрывай.
  
  ◈  Телефон не появляется в списке устройств —
     проверить USB-кабель (нужен data-cable, не зарядный),
     переустановить драйвер USB Android Composite.
  
  ◈  При запуске Hello Android — белый экран —
     подождать. Первый запуск приложения всегда дольше.
